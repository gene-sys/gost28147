






			НАЗНАЧЕНИЕ

	Программный эмулятор PC_DEC предназначен для исполнения DEC программ,
на персональных PC совместимых ЭВМ.



			ХАРАКТЕРИСТИКИ

	Программный эмулятор PC_DEC обеспечивает эмуляцию:

  процессора 1801 ВМ-3
  ОЗУ об'емом 56К
  страницу ввода/вывода пространства Q-BUS (при наличии адаптера PC Q-BUS)
  алфавитно цифровой дисплей 15-ИЭ-00-013  (в двух системах команд) или КЦГД
  гибкие диски в формате МY: (80 дорожек по 10 секторов)
  печатающее устройство  LP:
  таймер 50гц


                        УСЛОВИЯ ВЫПОЛНЕНИЯ

	Для работы программного эмулятора PC_DEC необходимо иметь:
  CPU не ниже  286
  RAM свободный об'емом не менее 192Кб без КЦГД
      свободный об'емом не менее 448Кб c эмуляцией КЦГД
  FDD 5.25" 1.2Mб/800Кб
  HDD об'емом для размещения программного эмулятора PC_DEC не менее 500Кб
      (без учета файлового хозяйства пользователя)



			ЗАПУСК ПРОГРАММЫ

	Вызов программы осуществляется средствами MS DOS.
	Возможны следующие способы запуска программы:

	     PC_DEC.EXE
	     Аналог включения питания и переход в пультовой режим.

	     PC_DEC.EXE filename
	     Загрузка файла filename в ОЗУ эмулятора с адреса 000000 и
	    передача управления по адресу 000000.

  	        Например:
	     PC_DEC.EXE rt11sj.sav r macro p81:prog.mac' exit
	     Загрузка монитора rt11sj.sav в ОЗУ эмулятора с адреса 000000 и
	     запуск в среде RT11SJ трансляции программы PROG1.MAC на
             макроассемблере MACRO, после окончания выход в среду MS-DOS.

	     PC_DEC.EXE rt11sj.sav r @py1:mack.com' exit
	     Загрузка монитора rt11sj.sav в ОЗУ эмулятора с адреса 000000 и
	     запуск в среде RT11SJ пакетного файла mack.com, содержащего
             комманды ОС RT11SJ, после окончания выход в среду MS-DOS.

             Примечание. Символ (') иммитирует нажитие на клавишу "ВК".


	     PC_DEC.EXE PY.BOT n
             Эта команда вызывает загрузку операционной системы RT11SJ
	    из файла образа системного диска устройства PYn:.
	    Где n один из символов:<пробел>,<Enter> или цифра от "0" до "7".


	В образ системного диска включены следующие драйверы:

     TT.SYS драйвер консольного терминала (177560-177566)
     MY.SYS драйвер гибкиого диска (MY0: дисковод A MS DOS)
				   (MY1: дисковод B MS DOS)
     LP.SYS драйвер печатающего устройства
     PY.SYS драйвер дисков.Каждое из устройств PY0: - PY7: связано с файлом
	    MS DOS. Содржимое файлов трактуется как диск в формате RT11.
  PC.SYS и  драйверы директорий MS DOS. Доступ к файлам директорий MS DOS
  P8.SYS    осуществляется командами определяющими полные имена файлов,
	    причем, имена файла должны удовлетворять синтасису имени файла
	    RT11 (не более 6-ти символов из набора A,B,C,..,Y,Z,0,1,..,9).
	    Директории устройств P80: - P87: совпадают с PC0: - PC7:
	    При обмене с устройствами PC0: - PC7: информация передается без
	    изменений (байт в байт).
	    При обмене с устройствами P80: - P87: информация перекодируется
	    из кода КОИ7 <-> КОИ8:
	    При зариси на P80: - P87:  КОИ7 -> КОИ8
	    При чтении с  P80: - P87:  КОИ8 -> КОИ7



	        Примеры:

	       COPY PC1:VFCRO.SAV MY0:
	       Перепись файла VFCRO.SAV из директории MS DOS на гибкий
	       диск в формате DEC без преобразоивния (байт в байт).

	       TYPE P80:VFCRO
	       Вывод на дисрлей файла VFCRO.LST содержащего информацию в КОИ8.

	       TYPE PC0:VFCRO.TXT
	       Вывод на дисрлей файла VFCRO.TXT содержащего информацию в КОИ7.

	       R MACRO
	       PC0:PROG,P80:PROG=P80:PROG
	       ^C
	       Трансляция файла PROG.MAC из директории MS DOS.
	       Получение  файла PROG.OBJ на устройстве PC0:.
	       Запись файла PROG.LST в директорию MS DOS.

     Примечание:
       Конфигурация файлов (связанных с устройствами PY0: - PY7:) и
       директорий (связанных с устройствами PC0: - PC7:) задается в
       файле RT11SJ.INI.Файл RT11SJ.INI берется из директории диска C: MS DOS
       в момент запуска программы PC_DEC.EXE.Файл RT11SJ.INI должен содержать
       ровно 16 строк, причем первые 8 строк должны содержать полные имена
       файлов (включая пути доступа) связываемых с устройствами PY0: - PY7:
       отсутствие пути доступа в строке трактуется как текущая директория.
       Следующие 8 строк должны содержать имена путуй доступа к директориям
       связываемых с устройствами PC0: - PC7: и P80: - P87:.Пустая строка
       задает текущую директорию.Каждая строка должна заканчмваться сразу за
       последним символом имени.

  Нажатие клавиши F2 вызывает переход в пультовой режим.
  Нажатие клавиши F3 вызывает включение/выключение таймера эмулятора PC_DEC.
  Нажатие клавиши F4 вызывает принудительное переключение на латинский регистр
  Клавише Esc    соответствует восьмеричный код 33 (АР2)
  Клавише F9     соответствует восьмеричный код 10 (ВШ)
  Клавише F10    соответствует восьмеричный код 12 (ПС)
  Клавише Insert соответствует восьмеричный код 33 120 (ПФ1)
  Клавише Delete соответствует восьмеричный код 33 122 (ПФ2)

  Для завершения работы эмулятора PC_DEC неоходимо выполнить программу
  EXIT.SAV или нажать клавишу F8.

			КОМАНДЫ ПУЛЬТОВОГО РЕЖИМА
Символ
0 до 7         - всьмеричная цифра
/              - открыть текущий адрес и прочитать слово по этому адресу
\ или Enter    - записать слово по текущему адресу
F11            - открыть текущий адрес плюс  2 и прочитать слово
Z или ^        - открыть текущий адрес минус 2 и прочитать слово
@ или .        - открыть адрес по содержимому  и прочитать слово
R (0-7,S)      - открыть регистр
P              - шаг
G              - пуск
S              - сохранить содержимое памяти (64k) в файле RT11SJ.SAV
L              - загрузить систему с устройства PY0
B xxx          - загрузить систему с устройства xxx
		 xxx   PY0 до PY7,MY0,MY1
